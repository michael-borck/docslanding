# Theme Usage Guide

## 🤖 **Script-Generated Landing Pages + Auto-Documentation**

This theme is designed for projects that:
1. **Generate landing pages** from README.md using scripts
2. **Want automatic documentation** from `docs/` folder structure
3. **Need consistent design** across multiple repositories

## 📁 **Required Structure**

```
your-repo/
├── _config.yml              # Theme configuration
├── index.md                 # Generated by your script from README.md
└── docs/
    ├── index.md             # Main docs page
    ├── getting-started/
    │   ├── index.md         # REQUIRED for auto-nav
    │   ├── installation.md
    │   └── quick-start.md
    ├── user-guide/
    │   ├── index.md         # REQUIRED for auto-nav
    │   └── settings.md
    └── features/
        ├── index.md         # REQUIRED for auto-nav
        └── feature-name.md
```

## ⚙️ **Minimal Configuration**

### 1. Essential _config.yml

```yaml
# Basic project info
title: Your Project Name
description: Your project description
baseurl: "/your-repo-name"
url: "https://your-username.github.io"

# GitHub info
github_username: your-username
repository: your-repo-name

# Theme settings
remote_theme: your-username/your-theme-name
auto_docs_navigation: true

# Optional: Customize header navigation (landing page only)
landing_navigation:
  - text: "Features"
    url: "#features"
  - text: "Installation"
    url: "#installation"
  - text: "Documentation"
    url: "/docs/"

# Optional: Customize header button
header_button:
  text: "GitHub"
  url: "https://github.com/your-username/your-repo-name"
```

## 🤖 **Script Integration**

### Your Script Should:

1. **Parse README.md** for project information
2. **Generate HTML content** using Tailwind CSS classes
3. **Create index.md** with Jekyll frontmatter + HTML body

### Generated index.md Template:

```markdown
---
layout: default
title: Project Name from README
description: Description from README
---

<!-- Your script generates this HTML from README.md -->
<section class="py-20 sm:py-24 lg:py-32">

        <h1 class="text-4xl sm:text-5xl lg:text-6xl font-extrabold text-stone-900 dark:text-white tracking-tight">{{ project_name }}</h1>
        
        <!-- Fixed button patterns your script can generate -->
        <div class="mt-10 flex flex-wrap justify-center gap-4">
            <a href="#features" class="bg-sky-600 text-white px-8 py-3 rounded-lg hover:bg-sky-700">Get Started</a>

            <a href="https://github.com/{{ username }}/{{ repo }}" class="bg-stone-700 text-white px-6 py-3 rounded-lg">GitHub</a>
            <a href="https://gitingest.com/{{ username }}/{{ repo }}" class="bg-sky-700 text-white px-6 py-3 rounded-lg">Ingest</a>

<!-- More sections generated from README content -->
```

## 📚 **Documentation Auto-Generation**

### How It Works:

1. **Jekyll scans** `docs/` folder for any `*/index.md` files
2. **Converts folder names** to navigation items:
   - `getting-started/index.md` → "Getting Started"
   - `user-guide/index.md` → "User Guide"
   - `api-reference/index.md` → "Api Reference"

3. **Generates navigation menu** automatically
4. **You link pages** manually within each section's `index.md`

### Section Index Template:

```markdown
---
layout: page
title: Getting Started
description: Get up and running quickly
---

# Getting Started

Welcome to this section...

## Pages in this section:
- [Installation](installation/) ← Links to installation.md
- [Quick Start](quick-start/)   ← Links to quick-start.md
```

## 🎨 **Styling**

### Available Classes:
The theme provides Tailwind CSS with custom colors:
- **Primary**: `bg-sky-600`, `text-sky-600`
- **Secondary**: `bg-stone-700`, `text-stone-600`
- **Success**: `bg-green-600`
- **Dark mode**: All classes have `dark:` variants

### Section Patterns:
```html
<!-- Hero section -->
<section class="py-20 sm:py-24 lg:py-32">
    <!-- Hero content -->
</section>

<!-- Alternating backgrounds -->
<section class="py-16 bg-white dark:bg-stone-800">
    <!-- Content -->
</section>

<section class="py-16 bg-stone-50 dark:bg-stone-900">
    <!-- Content -->
</section>
```

## 🔧 **Script Requirements**

Your script needs to generate:

### Required Elements:
1. **Jekyll frontmatter** with title/description
2. **Hero section** with project name
3. **Button row** with consistent styling
4. **Content sections** with proper IDs for navigation

### Button URLs (Fixed Patterns):
- Documentation: `/docs/`
- GitHub: `https://github.com/{{ username }}/{{ repo }}`
- Ingest: `https://gitingest.com/{{ username }}/{{ repo }}`
- Deep Wiki: `https://deepwiki.com/{{ username }}/{{ repo }}`

### Navigation Anchors:
Your script should generate sections with IDs that match your `landing_navigation`:
- `#features`
- `#installation`
- `#getting-started`
- etc.

## 🚀 **Deployment**

1. **Script generates** `index.md` from README.md
2. **Manual setup** of `docs/` structure with `index.md` files
3. **Enable GitHub Pages** (Deploy from branch: main, folder: root)
4. **Theme auto-handles** navigation, styling, docs integration

## 🐛 **Common Issues**

### Docs Navigation Not Showing
- Ensure each docs folder has an `index.md` file
- Check `auto_docs_navigation: true` in `_config.yml`

### Buttons Not Styled
- Verify Tailwind classes in generated HTML
- Check Jekyll is processing the `index.md` (not serving static HTML)

### 404 Errors
- Verify `baseurl` matches repository name
- Check file paths and folder structure match examples
